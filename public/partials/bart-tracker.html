BART Tracker

I originally started this project for my personal use. Since I am a BART commuter, I wanted an app that can quickly retrieve real-time info without using a lot of bandwidth. So I decided to use JQuery Mobile for the UI and HTML5 Local Storage for storing favorite routes. I also compiled all the html, css, and js files into the app to minimize bandwidth usage. The only time this app connects to the web is for Ajax calls to bart.gov to retrieve information.

When I first started this project I was using Phone Gap for porting the app. But I noticed the slow startup time that came with using Phone Gap, so I manually ported the app to each platform by using web views. Using web views turns out to have a huge performance gain. Startup time went from about 7 seconds to about 2 seconds on a Samsung Galaxy S3 running Android 4.
